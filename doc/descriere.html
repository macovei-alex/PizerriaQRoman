Descriere

Aspecte generale

Proiectul reprezinta un sistem complet de comandare, receptionare a comenzilor, si administrare a unui meniu de restaurant pizzerie. Restaurantul este amplasat fizic in orasul Roman, judetul Neamt. Site-ul se poate vizita dand clic aici. Sistemul este, in mare, alcatuit din 4 aplicatii diferite:
  1. Aplicatia server
  2. Aplicatia de comanda a clientului
  3. Aplicatia de primire a comenzilor pentru casier
  4. Aplicatia de administrare a sistemului
Sistemul nu va trata livrarea comenzilor, aceasta fiind oferita de catre un tert. In functie de API-ul pus la dispozitie de catre acesta exista posibilitate integrarii unei harti in timp real pentru urmarirea livrarii.

Aplicatia server

Este o aplicatie Java Spring ce foloseste framework-urile Spring Boot pentru a oferi servicii REST si Spring Data JPA pentru conexiunea la baza de date MySQL. Securitatea utilizatorului va fi oferita de Spring Security. Aplicatia este responsabila cu gestionarea datelor despre utilizatori, meniu, comenzi. Pentru modelul de date am pus accent pe pastrarea unui istoric cat mai complet al comenzilor.

Aplicatia de comanda a clientului

Este o aplicatie pentru mobil scrisa in React Native. Aceasta aplicatie doreste sa aiba un design cat mai simplu si accesibil, dar fara sa sacrifice din functionalitate. Puncte cheie le reprezinta posibilitatea de creare conturilor folosind conturi Google sau adrese de email, asocierea conturilor cu adrese de livrare, vizualizarea istoricului de comenzi si a meniului restaurantului.

Aplicatia de receptie a comenzilor de catre casier

Va fi o aplicatie React Native scrisa cu scopul de a fi distribuita doar pe tablete Android. Scopul acesteia este de a vizualiza comenzile active, a oferi un timp estimat de asteptare, si a seta statusurile comenzilor.

Aplicatia de administrare a sistemului

Va fi o aplicatie web scrisa (cel mai probabil) in React. Aceasta aplicatie va fi folosita de catre administratorii restaurantului pentru a modifica meniul, optiunile fiecarui produs, aranjamentul produselor pe interfata grafica a clientilor si pentru a vizualiza statistici despre clienti si a oferi discounturi.

Obiectivul proiectului pana la termenul de predare

Pana la termenul de predare, sistemul trebuie sa contina:
  • un server functional, care sa nu crape
  • un model de date aproape complet (cu exceptia datelor legate de utilizator) si o baza de date populata cu date de test
  • o aplicatie client prin care sa se poata efectua si vizualiza comenzi
  • o aplicatie client cu interfata care sa se conforme pe designul propus in prealabil*
Aspecte care vor fi dezvoltate dupa termenul de predare:
  • securitatea datelor
  • tranzactionarea cu valuta reala
  • aplicatia casierului
  • aplicatia administratorului
* Designul propus in prealabil este un prototip creat in Figma pentru validare functionala si referinta vizuala pentru aplicatia clientului.
